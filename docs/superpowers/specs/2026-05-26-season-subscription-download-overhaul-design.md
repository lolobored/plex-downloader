# Season Subscription + Download UX Overhaul — Design

## Goal

Fix three related UX problems:
1. Subscribing from the Season Detail page incorrectly creates a show-level subscription. It should create a season-level subscription (keep N unwatched episodes ahead, scoped to that season only).
2. "One-off download" (now renamed "Download") should enqueue ALL episodes of the scoped show/season — not just the next N unwatched.
3. The episode-level view must not offer a subscribe option — only a direct download button (already the case via `DownloadButton`; must stay that way).

---

## Backend

### New DB table: `season_subscriptions` (Liquibase migration `010-season-subscriptions.yaml`)

```sql
CREATE TABLE season_subscriptions (
  id           BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id      BIGINT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  season_id    BIGINT NOT NULL REFERENCES seasons(id) ON DELETE CASCADE,
  target_count INT NOT NULL CHECK (target_count IN (5, 10, 15, 20)),
  created_at   TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT now(),
  updated_at   TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT now(),
  CONSTRAINT uq_user_season_sub UNIQUE (user_id, season_id)
);
```

### New entity: `SeasonSubscription`

Fields: `id`, `user` (ManyToOne), `season` (ManyToOne), `targetCount`, `createdAt`, `updatedAt`.
Unique constraint: `(user_id, season_id)`.

### New repository: `SeasonSubscriptionRepository`

Methods needed:
- `findByUserId(Long userId) → List<SeasonSubscription>`
- `findByUserIdAndSeasonId(Long userId, Long seasonId) → Optional<SeasonSubscription>`
- `findAllWithUserAndSeason() → List<SeasonSubscription>` (join-fetched, for `WatchedSyncService.syncAll()`)
- `findActiveEpisodeIdsForSeason(Long userId, Long seasonId) → Set<Long>` — active queue items scoped to a season

### New DTO: `SeasonSubscriptionResponse`

Record: `seasonId`, `showId`, `seasonTitle`, `targetCount`, `updatedAt`.
Static factory `from(SeasonSubscription sub)`.

### New DTO: `SeasonSubscriptionRequest`

Record: `seasonId`, `targetCount`.

### `SubscriptionService` additions

```
upsertSeason(userId, seasonId, targetCount) → SeasonSubscriptionResponse
  - load user, load season (which implies show via FK)
  - upsert SeasonSubscription
  - return SeasonSubscriptionResponse.from(sub)

cancelSeason(userId, seasonId)
  - find sub; if present: delete + cancelAllForSeason(userId, seasonId)

replenishSeason(SeasonSubscription sub)
  - watchedIds  = watchedRepo.findWatchedEpisodeIds(userId, showId)
  - activeIds   = queueRepo.findActiveEpisodeIdsForSeason(userId, seasonId)
  - activeUnwatched = activeIds - watchedIds
  - deficit = targetCount - activeUnwatched
  - if deficit <= 0: return
  - next episodes: episodeRepo.findBySeasonIdOrderByEpisodeNumber(seasonId) filtered by !watched && !active
  - enqueue up to deficit

replenishIfSubscribedSeason(userId, seasonId)
  - findByUserIdAndSeasonId → ifPresent replenishSeason
```

### `DownloadService` addition

```
cancelAllForSeason(Long userId, Long seasonId)
  - cancel all PENDING/IN_PROGRESS queue items for episodes in this season belonging to this user
```

### `DownloadQueueRepository` addition

```
findActiveEpisodeIdsForSeason(Long userId, Long seasonId) → Set<Long>
  - query: items where user=userId, mediaType=EPISODE, mediaId in episode IDs of that season, status IN (PENDING, IN_PROGRESS)
```

### `SubscriptionController` additions

New endpoints (all under `/api/subscriptions`):

```
GET  /api/subscriptions/seasons
  → List<SeasonSubscriptionResponse> for authenticated user

POST /api/subscriptions/seasons
  body: SeasonSubscriptionRequest {seasonId, targetCount}
  → SeasonSubscriptionResponse
  → also calls syncShow(userId, showId) + replenishIfSubscribedSeason(userId, seasonId)

DELETE /api/subscriptions/seasons/{seasonId}
  → 204 No Content

GET  /api/subscriptions/seasons/{seasonId}/queue-count
  → {count: N}
```

### Remove endpoint

`POST /api/download/show/{showId}/unwatched` — removed from `DownloadController`. The "Download all" action now uses the existing generic `POST /api/download` endpoint with `type=SEASON` or `type=SHOW`, which already enqueues all episodes.

### `WatchedSyncService.syncAll()` addition

After the show-subscription loop, add a season-subscription loop:

```java
seasonSubscriptionRepo.findAllWithUserAndSeason().forEach(sub -> {
    try {
        Long showId = sub.getSeason().getShow().getId();
        syncShow(sub.getUser().getId(), showId);
        subscriptionService.replenishSeason(sub);
    } catch (Exception e) {
        log.error("Season watched sync failed for user={} season={}: {}",
            sub.getUser().getId(), sub.getSeason().getId(), e.getMessage());
    }
});
```

## Frontend

### `api/watched.js` additions

```js
export const getSeasonSubscriptions  = () => http.get('/api/subscriptions/seasons').then(r => r.data)
export const subscribeSeason         = (seasonId, targetCount) =>
  http.post('/api/subscriptions/seasons', { seasonId, targetCount }).then(r => r.data)
export const unsubscribeSeasonSub    = (seasonId) =>
  http.delete(`/api/subscriptions/seasons/${seasonId}`)
export const getSeasonQueueCount     = (seasonId) =>
  http.get(`/api/subscriptions/seasons/${seasonId}/queue-count`).then(r => r.data.count)
```

Remove `enqueueUnwatched` export (endpoint deleted).

### `stores/watched.js` additions

```js
const seasonSubscriptions = ref(new Map())  // Map<seasonId, targetCount>

// fetchSubscriptions() also calls getSeasonSubscriptions() and populates seasonSubscriptions

async function subscribeSeason(seasonId, targetCount) {
  await apiSubscribeSeason(seasonId, targetCount)
  seasonSubscriptions.value.set(seasonId, targetCount)
}

async function unsubscribeSeason(seasonId) {
  await apiUnsubscribeSeasonSub(seasonId)
  seasonSubscriptions.value.delete(seasonId)
}

function getSeasonSubscription(seasonId) {
  return seasonSubscriptions.value.get(seasonId) ?? null
}
```

Remove `enqueueUnwatched` action.

### `SubscribeButton.vue` changes

**Props:** Add `seasonId: { type: Number, default: null }`.

**Computed `current`:**
- If `seasonId` set → `watchedStore.getSeasonSubscription(seasonId)`
- Else → `watchedStore.getSubscription(showId)`

**`doSubscribe(n)`:**
- If `seasonId` set → `watchedStore.subscribeSeason(seasonId, n)`
- Else → `watchedStore.subscribe(showId, n)` (existing)

**`doDownloadAll()`** (replaces `doOneTime(n)`):
- If `seasonId` set → `POST /api/download` `{type:'SEASON', id:seasonId}`
- Else → `POST /api/download` `{type:'SHOW', id:showId}`
- Single button, no N-picker

**`handleUnsubscribeClick()`:**
- If `seasonId` set → `getSeasonQueueCount(seasonId)` + `watchedStore.unsubscribeSeason(seasonId)`
- Else → `getShowQueueCount(showId)` + `watchedStore.unsubscribe(showId)` (existing)

**Template picker — "One-time download" section replaced:**
```html
<div class="picker-section">
  <p class="picker-label">Download</p>
  <button class="picker-opt" @click="doDownloadAll">⬇ Download all</button>
</div>
```

### `TvShowDetailView.vue`

Remove `<SubscribeButton :showId="show.id" small />` from inside the season `PosterCard` `#badge` slot. Season cards show no subscribe badge.

### `SeasonDetailView.vue`

Change:
```html
<SubscribeButton :showId="show.id" />
```
To:
```html
<SubscribeButton :showId="show.id" :seasonId="season.id" />
```

## What does NOT change

- `ShowSubscription` model and `show_subscriptions` table — untouched.
- Show-level subscription flow — untouched.
- Episode-level UI — `DownloadButton` only, no `SubscribeButton`. Already correct; nothing to change.
- `POST /api/download` (type=SEASON/SHOW/EPISODE/MOVIE) — already enqueues all episodes; no changes needed.
